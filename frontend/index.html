Research

Research Questions & Goals

Three empirical questions guide evaluation of the system's core capabilities.

RQ1

Classification Accuracy

How accurately can fine-tuned DeBERTa-v3 classify sustainability news compared to zero-shot and keyword baselines on real GDELT data?

🎯 Target: F1 ≥ 0.80 on 300-article labeled test set
RQ2

RAG Summary Quality

Can a RAG-based system generate coherent, source-grounded trend summaries from GDELT abstracts for a specified theme, region, and time window?

🎯 Target: G-Eval relevance + coherence ≥ 4.0 / 5.0
RQ3

Hybrid Retrieval Gain

Does hybrid retrieval (text-embedding-3-small + BM25 keyword) outperform single-mode retrieval for sustainability domain queries?

🎯 Target: MRR & precision@5 improvement over dense-only
Methodology

Data & Models

Open-source data and carefully budgeted OpenAI calls form a reproducible, evaluable system.

🗃️ Data Sources

  • 📡
    GDELT ProjectFree global dataset — millions of tagged news events fetched via Doc API v2
  • ✏️
    Abstract only (~100 tokens/article)Titles + snippets; keeps token costs under $0.05 for the full pipeline
  • 📄
    IPCC PDFsGround truth factual source for RAG hallucination evaluation
  • 💾
    SQLite local DBStores articles, embeddings (JSON), sentiment scores; no cloud required

🤖 Models & Tools

  • 🔗
    text-embedding-3-small1536-dim embeddings at $0.02/1M tokens — cheapest OpenAI embedder
  • 🎭
    GPT-3.5-turbo (sentiment)Batched sentiment scoring, 5 articles per call, ~$0.01 total for 100 articles
  • ⚡
    GPT-3.5-turbo (RAG)Summarisation from retrieved abstracts; max 220 tokens output per query
  • 📐
    Cosine similarity (pure Python)No FAISS needed for 100 articles; numpy-free cosine over stored vectors
